How files are tagged

`Files` table

One row for each file indexed by hooYa

Id

This is an MD5 hash of the file; this is a fast way to organize files. Typically, a user supplies the MD5 hash of the file s/he wants and hooYa looks up that hash and serves w/e content is asked for.

Path

Path where the indexed file can be accessed

Size

Filesize in bytes

Class

Type of media. Supported values are anime, single_image, and video but adding a new class only involves writing the code to render it.

Mimetype

Content type determined by your OS's magic.mime file

`TagMap` Table

One row for each file's tag

FileId

MD5 hash of the file that this tag describes. This can describe at most one (1) entry in `Files`

Added

Date that the tag was added. Not used for anything but cute to have

TagId

Selects which Namespace -> Member pair this tag describes. This can describe at most one (1) entry in `Tags`.

`Tags` Table

One row for each Namespace -> Member pair. These are pairs like 'series' -> 'puella_magi_madoka_magica'

Id

Primary Key for each pair

Space

Describes a namespace, such as series, character, or some other similar category. These spaces are not static, so feel free to add as many spaces as you want!

Member

Describes a member in a namespace, such as puella_magi_madoka_magica or kaname_madoka
